# AGENTS.md

> 參考 Karpathy 的 Agent 行為規範，配合 Yan 的工作習慣定制

## 核心原則

### 1. 思考後再行動，不懂就問
- 先一步一步思考再動手
- 有任何模糊不清的地方，馬上問清楚
- 永遠不要假設

### 2. 簡單優先，拒絕過度工程
- 寫最簡單能用的方案
- **每行代碼只做一件事**
- 50行清晰代碼 > 10行複雜「優雅」代碼
- 避免不必要的抽象和設計模式

### 3. 精準下手，不碰無關代碼
- 只改任務相關的檔案
- 永遠不要「順手」優化、整理或重構不相關的代碼
- 完美配合現有的風格和架構

### 4. 目標導向 + 驗證
- 先定義「完成」是什麼樣子
- 小改動 → 驗證 → 再調整
- 用最小的迭代完成任務

## 適用場景
- 所有軟件開發任務
- 代碼審查
- 新功能規劃
- 系統設計
